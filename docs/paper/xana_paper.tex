\begin{abstract}
\noindent
We present \textbf{XANA}, an open-source personal cognitive AI runtime that fuses four previously
separate research threads into a single deployable system: (1)~\emph{neuro-symbolic reasoning}, where
a Rust CLIPS-pattern forward-chaining engine verifies and augments LLM outputs; (2)~\emph{adversarial
evolutionary optimization} via a Digital Red Queen tournament~\cite{kumar2025redqueen} that continuously
evolves typed cognitive algorithms in a virtual machine; (3)~\emph{fuzzy emotional modulation}
grounded in Zadeh's formal inference framework~\cite{zadeh1965fuzzysets,zadeh1975fuzzylogic}, which
adapts system behavior to affective context; and (4)~\emph{security-by-architecture}, where a Rust
middleware layer enforces PII detection and prompt-injection rejection at $2.1\,\mu$s per call ---
structurally, not as a Python-level filter. A Rust Kalman filter ($1.4\,\mu$s via PyO3 zero-copy
buffers) tracks epistemic uncertainty before facts enter the symbolic engine. Five specialized agents
(the \emph{Apex Quintet}) communicate through a typed AION protocol that eliminates hallucination
cascade. We evaluate XANA using the \textbf{XANA Fitness Index} (XFI), a seven-pillar composite
benchmark. With a full Docker infrastructure stack, XANA achieves $\mathrm{XFI} = 100.0/100$.
The complete system --- Rust, Python, TypeScript, Docker, benchmark suite --- is available at
\url{https://github.com/kemquiros/zana-core} under the MIT License.
\bigskip\\
\noindent\textbf{Keywords:} cognitive architecture, neuro-symbolic AI, local-first AI, adversarial
evolutionary optimization, fuzzy logic, Kalman filter, multi-agent systems, Rust, open source.
\end{abstract}

\section{Introduction}
% ===========================================================

The dominant paradigm for building AI assistants is shallow: prompt a foundation model, parse the
output, call it done. This approach produces systems that appear capable but are epistemically fragile.
They cannot explain their inferences, lose context between sessions, and offer no resistance to
adversarial inputs that exploit the absence of a verification layer.

A \emph{personal cognitive AI} intended to augment an individual's long-term reasoning and memory
requires a different architecture. We identify six requirements:

\begin{enumerate}[leftmargin=*, label=\textbf{R\arabic*.}]
  \item \textbf{Explicit memory separation.} Episodic, semantic, procedural, and world-model stores must
    be distinct --- conflating them produces false associations and inconsistent recall.
  \item \textbf{Symbolic verification.} A deterministic rule engine must form a second verification pass
    over probabilistic LLM outputs.
  \item \textbf{Signal uncertainty tracking.} Sensory observations must be noise-filtered before being
    asserted as facts in the symbolic engine.
  \item \textbf{Security-by-architecture.} PII detection and injection rejection must be structural, not
    optional middleware.
  \item \textbf{Evolutionary self-improvement.} The system should be able to improve its own cognitive
    algorithms through adversarial competition, not only gradient descent on fixed weights.
  \item \textbf{Emotional modulation.} Affective state affects attention allocation and decision
    confidence in biological cognition~\cite{bates1994emotion}; a personal AI must account for it.
\end{enumerate}

XANA satisfies all six requirements. Figure~\ref{fig:arch} shows the complete layered architecture.

The filter tracks a scalar signal-quality state. The zero-copy buffer path avoids
Python--Rust allocation overhead and is $4.6\times$ faster than the list path:

\begin{lstlisting}[language=Python, caption={PyO3 buffer vs list path}]
# Zero-copy buffer path (1.4 us/call)
kalman.update_buffer(obs_np_array)   # no allocation

# Python list path (6.5 us/call — compatibility only)
kalman.update([v1, v2, v3])
\end{lstlisting}

\subsubsection{Policy Brain and EML Operator}

The Policy Brain is a $384\!\to\!64\!\to\!4$ feedforward network with 8-accumulator FMA unrolling
(hides the 5-cycle FMA latency on x86) and row-major weight layout (ensures contiguous cache access).
Achieved latency: $7$--$10\,\mu$s. The EML (Exponential-Log) operator provides a numerically
stable identity $\log(\exp(x)) \equiv x$ \cite{odrzywolekEML}, used for round-trip signal
verification; error = $0.0$.

\subsection{Fuzzy Heart: Emotional Modulation}

Inspired by Bates~\cite{bates1994emotion} and grounded in Zadeh's formal
framework~\cite{zadeh1965fuzzysets,zadeh1975fuzzylogic}, the Fuzzy Heart maps three scalar inputs ---
surprise $s$, valence $v$, arousal $a \in [0,1]$ --- to a mood vector $\mathbf{m} \in [0,1]^4$
via Mamdani inference~\cite{mamdani1974application}:

\begin{align}
  \mu_{\text{alert}}(s,v) &= \min\!\left(\mu_\text{HIGH}(s),\, \mu_\text{NEG}(v)\right) \\
  T &= T_0 + 0.3\,m_{\text{alert}} - 0.2\,m_{\text{calm}}
\end{align}

The defuzzified temperature offset $\delta T$ modulates LLM sampling and reasoning depth.

\subsection{Multimodal Gateway Pipeline}

\begin{algorithm}[H]
\caption{Multimodal Gateway Request Pipeline}
\begin{algorithmic}[1]
\Require Raw input $x$ (text $\vee$ audio $\vee$ image $\vee$ multimodal)
\State $x' \leftarrow \mathrm{modality\_processor}(x)$
  \Comment{Whisper / LLaVA-Claude / text}
\State $\mathrm{Armor.inspect\_input}(x')$
  \Comment{$2.1\,\mu$s; \textbf{abort} $\to$ HTTP 403 on reject}
\State $\hat{x}, P \leftarrow \mathrm{KalmanFilter.update\_buffer}(x')$
  \Comment{$1.4\,\mu$s confidence estimate}
\State $\mathbf{m} \leftarrow \mathrm{FuzzyHeart.infer}(\hat{x})$
  \Comment{mood vector, adjusts $T$}
\State $\mathcal{M} \leftarrow \mathrm{MemoryQuery}(\hat{x})$
  \Comment{ChromaDB + PostgreSQL via Symbiosis MCP}
\State $r \leftarrow \mathrm{LLM}(\hat{x}, \mathcal{M}, T(\mathbf{m}))$
  \Comment{temperature-adjusted inference}
\State $e \leftarrow \mathrm{PerceptionEvent}(x', \hat{x}, P, r)$
\State $e' \leftarrow \mathrm{ReasoningEngine.run}(e)$
  \Comment{CLIPS forward chaining to fixed point}
\State $\mathrm{Armor.inspect\_output}(e')$
  \Comment{\textbf{abort} $\to$ HTTP 403 on reject}
\State \Return $e'$
\end{algorithmic}
\end{algorithm}

\subsection{Reasoning Engine}

The ReasoningEngine implements CLIPS-inspired forward-chaining over typed \texttt{DefTemplate} facts.
The inference cycle runs until a fixed point (no new facts asserted):

\begin{lstlisting}[language={}, caption={CLIPS-pattern rule example (Rust DSL)}]
(defrule high-surprise-alert
  (PerceptionEvent (surprise ?s&:(> ?s 0.4))
                   (valence  ?v&:(< ?v 0.0)))
  =>
  (assert (MoodState (mood ALERT) (intensity ?s)))
  (assert (ReasoningDepth (depth DEEP))))
\end{lstlisting}

Dynamic rules from swarm peers pass through \texttt{LLMGuard} before loading --- preventing rule
injection attacks (Milestone~8.4).

Inspired by the Digital Red Queen paper~\cite{kumar2025redqueen} and AutoML-Zero~\cite{real2020automlzero},
\emph{Idle Zero} represents each cognitive algorithm as a typed \texttt{WarriorAlgorithm}:

\begin{lstlisting}[language=Rust, caption={WarriorAlgorithm data structure}]
pub struct WarriorAlgorithm {
    pub id:         String,
    pub generation: u32,
    pub setup:      Vec<Instruction>,   // initialization
    pub predict:    Vec<Instruction>,   // inference
    pub learn:      Vec<Instruction>,   // online update
    pub fitness:    f64,
}
\end{lstlisting}

Instructions operate over a 64-slot memory array using the EML-derived minimal instruction set:
\texttt{\{NoOp, Add, Sub, Mul, Div, Exp, Log, Sin, Cos, GaussianSample, EmulativeCall\}}~\cite{odrzywolekEML}.
Fitness is evaluated on real personal data (\texttt{real\_data.txt}), embedding curriculum
principles~\cite{bengio2009curriculum} by progressively increasing data difficulty as generations advance.

\section{Related Work}
% ===========================================================

\subsection{Cognitive Architectures}
Classical architectures ACT-R~\cite{anderson2004actr} and SOAR~\cite{laird2012soar} integrate
symbolic and procedural knowledge but require hand-crafted rules and predate modern neural models.
XANA uses the symbolic engine as a \emph{verification layer} over LLM outputs rather than as the
primary reasoning substrate.

\subsection{Fuzzy Logic}
Zadeh~\cite{zadeh1965fuzzysets} introduced continuous $[0,1]$ membership. The computational framework
of linguistic variables and IF-THEN rules followed in~\cite{zadeh1973outline,zadeh1975fuzzylogic}.
Mamdani~\cite{mamdani1974application} provided the first practical control architecture; Takagi and
Sugeno~\cite{takagi1985fuzzy} extended it to ML-compatible output forms. XANA uses Mamdani inference
for the Fuzzy Heart.

\subsection{World Models}
Ha and Schmidhuber~\cite{ha2018worldmodels} showed that an agent can learn a compressed world model
and train inside its own ``dream.'' XANA's Neo4j world model plays an analogous role, enabling
multi-hop symbolic reasoning over a graph of entities extracted from episodic experience.

\subsection{Reinforcement and Curriculum Learning}
Procedural memory Q-values follow IRL intuition~\cite{ng2000irl}. Idle Zero uses curriculum
principles~\cite{bengio2009curriculum} by increasing training difficulty as warrior fitness improves.

\subsection{Evolutionary Machine Learning}
AutoML-Zero~\cite{real2020automlzero} showed that full ML algorithms can be evolved from primitives.
XANA's Idle Zero extends this: warriors are typed programs in a cognitive VM, and evolution uses
adversarial Red Queen pressure~\cite{kumar2025redqueen} rather than standard tournament selection alone.
Model merging~\cite{akiba2024modelmerging} and preference optimization~\cite{lu2024preferenceoptalgo}
inform the roadmap for personalizing local LLM weights.

\subsection{LLM Agents and Security}
LangGraph~\cite{langchain2024langgraph} and smolagents~\cite{huggingface2024smolagents} provide the
execution backbone for the Apex Quintet. XANA wraps them with the AION typed protocol. Prompt
injection~\cite{greshake2023promptinjection} motivates the Armor Rust layer: making security
impossible to bypass through the Python application tier.

% ===========================================================
\section{Discussion}
% ===========================================================

\subsection{Limitations}

\textbf{Rust compilation.} Steel Core and Armor require Rust 1.78+ with AVX2. Users on ARM or
non-AVX2 x86 must remove \texttt{-C target-cpu=native}.

\textbf{Foundation model dependency.} Best reasoning quality requires a capable LLM (Claude Sonnet or
equivalent). The symbolic layer compensates but cannot fully substitute on resource-constrained hardware.

\textbf{Single-node deployment.} Distributed swarm reasoning (Milestone~8.3) is designed but not yet
implemented.

\textbf{Red Queen convergence time.} The 2000-generation tournament is CPU-intensive. \texttt{rayon}
parallelism mitigates this, but it remains unsuitable for embedded devices.

\subsection{Future Work}
\label{sec:future}

\textbf{Milestone 8.3 --- Remote Swarm Query.} Broadcast \texttt{RemoteQuery} to peer nodes when the
local engine has no rule for an observed fact. Peer responses are scored by LLMGuard before loading.

\textbf{Online Red Queen.} Integrate the tournament loop with live interaction data, combining
curriculum learning~\cite{bengio2009curriculum} with real-time feedback.

\textbf{LLM personalization via model merging.} Following~\cite{akiba2024modelmerging}, champion
warrior algorithms could guide low-rank adaptations of the local LLM weights, producing a
personalized model without full fine-tuning.

\textbf{Civic Ledger (Milestone 8.1).} Mirror the rule base to a SHA-256 audit log, making rule
changes publicly auditable.

% ===========================================================
\section{Conclusion}
% ===========================================================

XANA demonstrates that a personal cognitive AI can simultaneously satisfy epistemic rigor, security
guarantees, and practical deployability. The contributions are:

\begin{enumerate}[leftmargin=*, label=\arabic*.]
  \item A \textbf{seven-layer neuro-symbolic architecture} where Kalman filtering, fuzzy emotional
    modulation, LLM inference, and CLIPS-pattern reasoning operate as sequential enrichment and
    verification layers.
  \item \textbf{Security-by-architecture}: Rust Armor makes PII and injection checks impossible to
    bypass through the Python application layer, at $2.1\,\mu$s per call.
  \item A \textbf{Digital Red Queen tournament} (Idle Zero) that adversarially evolves typed cognitive
    algorithms and crystallizes champions back into the procedural memory registry.
  \item A \textbf{typed inter-agent protocol} (AION) that eliminates hallucination cascade in
    five-agent Apex Quintet pipelines.
  \item A \textbf{reproducible seven-pillar benchmark} (XFI) with a reference implementation
    achieving $100.0/100$ on a standard workstation.
\end{enumerate}

The complete system is available at \url{https://github.com/kemquiros/zana-core} (MIT License).
